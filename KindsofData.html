Kinds of Data

R has a very rich variety of kinds of data.  Note that we did not use the term data type, which can mean different things depending on the context and how precise the writer is. In addition, R has things that most people would consider data that R treats as second class, ie not like normal data..


The beginner can be confused by, eg,  the following statements:


  1. "R has no scalars"
  2. "Everything is an object"
  3. "Everything is a vector"
  4. "Column 3 contains variables"
  5. "a type of a variable"


But what do these terms mean?  What does " everything" refer to?  The problems rest in the imprecise use of terminology, and often also in conceptual misunderstanding. For example the official R documentation makes the first statement, but in the text the term "scalar" appears. This can't be misunderstanding; it shows the need exists for the concept, and why the authors of R talk about scalars and insist there are none is hard to say. Eventually one realizes what they mean.


R's design was influenced by its predecessor, S, intended for statistical numerical processing. This explains the decision to make the notions of vectors and matrices central. It was also influenced by the Lisp dialect Scheme, an elegant functional language, and can do anything Scheme can do. But the designers tried to add onto S many features that made R complex, a sort of Swiss Army knife, good for cutting but not so good as a hammer. It appeals to those who just want tools, but not to purists.


We cannot go into a complete discussion here, because that is covered in the many separate topics. But here is a classification that may help the beginner:


There are 2 kinds of Objects (the normal kind of data):

       Vectors

               Atomic: no structure except a linear order, elements being all the same "atomic type"

               Arrays: multi-dimensional, where each dimension is an atomic vector, again all the same type.

               Lists: a linear stucture of any kind, recursively

       Non-Vectors

               There are 19 of these (see Objects)


Though the term "atomic" is used heavily, it only applies to vectors. It is comprised of numbers, character strings, logicals (Booleans) and a few more unusual things like   NA, meaning a value that should be there but is not available.  The kinds of things in atomic vectors then are  what most people would call "scalars".


Thus nearly all things we would normally consider data are subsumed under Vectors, of which there are 3 kinds: Atomic Vectors, Arrays, and Lists. But we soon discover that things like the number 1, the string 'abc' and the boolean TRUE are all vectors! To create a vector the operator c() is used, analogous to Lisp's cons. But now things get more interesting.  1 and c(1) are identical. In other words, things we would consider scalars all live inside a vector. So 1+1=2 is really c(1) + c(1) = c(2). An so on. Whenever R needs to do an operation that applies to the elements of a vector it does just that.  This becomes very confusing for beginners, particularly with strings. Often R documentation conflates "string" with "a string vector", and in some contexts this can be tricky.


Other ways of distinguishing different kinds of data include type, mode, and class, depending on the kind of distinction. Mode and class are more general classifications, and,  like type,  determine what operations are appropriate.


However there is a category of data called  Levels that is used for classifying things in a non-numeric way, a bit like Booleans with many values. Factors  are "vectors of levels", because the levels are converted into integers. A factor f then has type = integer vector, but class=factor. The levels then are just names for the integers that appear in f. But you can't add two levels, because it makes no sense; they are intended as names, not numbers. Applying a factor function to f works because of its class, but you can also treat it as integers if you want to be perverse, though arithmetic with them would be meaningless. Classes thus are a higher level of abstraction, whose representation is some kind of object (its storage mode, a C data type). Until factors were introduced, people had to represent these data by other means, usually character strings. Factors make this more efficient. Some other languages have packages that deal with factors (Python has Pandas), but they are fundamental in R.
